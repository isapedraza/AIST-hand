# Documentation

Project documentation is grouped by purpose:

- `ARCHITECTURE.md` - current package and application boundaries.
- `MIGRATION.md` - path changes from the monorepo restructuring.
- `decisions/` - project decision log.
- `research/` - method notes, comparisons, validation examples, and paper drafts.
- `handoff/` - historical continuation notes from earlier work sessions.
- `archive/` - legacy root artifacts kept for traceability, not active workflows.
- `assets/` - images used by notes and reports.
- `TODO.md` - current project task notes.
