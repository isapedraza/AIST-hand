## El problema actual
El modelo 3-manos `stage1_best_total(15).pt` (6k steps, B=50k total) se mueve
pero con ROM muy limitada, mucho peor que Run20 (1 mano, 15k steps, metrica
madura). Comparar con Run20 es injusto: esto es el Run 1 de un setup mas dificil.

Factor dominante medido = **sub-entrenamiento ~7x**:
- Run20 shadow: 50k/step x 15k = ~750M updates de D_r.
- Este 3-manos shadow: 50k/3 = 16.6k/step x 6k = ~100M.
- En T4 NO se puede igualar el batch per-robot de Run20 con 3 manos (B=150k -> OOM).
  Se compensa con steps (~45k para igualar) o menos robots por run.
